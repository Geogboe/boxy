# Architecture Decision Records (ADRs)

This folder contains Architecture Decision Records for Boxy.

## Purpose

ADRs are used to record decisions that affect the project long-term:

- what we decided
- why we decided it
- what alternatives we considered
- what consequences/tradeoffs we accepted

These notes are meant to prevent "tribal knowledge" and re-litigating the same
design choices repeatedly.

## Conventions

- File naming: `NNNN-short-title.md` (NNNN is zero-padded)
- Status: `Proposed` | `Accepted` | `Deprecated` | `Superseded`
- Keep decisions concrete: include "correct usage" and invariants when relevant
