# Boxy Quick Start Guide

This guide will get you up and running with Boxy in minutes.

## Prerequisites

- **Docker**: Required for container provisioning
  - Linux: Install Docker Engine
  - macOS: Install Docker Desktop
  - Windows: Install Docker Desktop (with WSL2)
- **Go 1.21+** (if building from source)

## Installation

### Option 1: Install from Binary (Recommended)

Download the latest release for your platform:

```bash
# Linux / macOS (adjust version and platform as needed)
curl -L https://github.com/Geogboe/boxy/releases/latest/download/boxy-linux-amd64 -o boxy
chmod +x boxy
sudo mv boxy /usr/local/bin/

# Verify installation
boxy --version
```

### Option 2: Build from Source

```bash
git clone https://github.com/Geogboe/boxy.git
cd boxy
make build
sudo mv build/boxy /usr/local/bin/
```

### Option 3: Run with Docker Compose

```bash
git clone https://github.com/Geogboe/boxy.git
cd boxy
docker-compose up -d
```

## Initial Setup

### 1. Create Configuration File

Create `~/.config/boxy/boxy.yaml`:

```yaml
# Storage configuration
storage:
  type: sqlite
  path: ~/.config/boxy/boxy.db

# Logging configuration
logging:
  level: info
  format: text

# Define your resource pools
pools:
  - name: ubuntu-pool
    type: container
    provider: docker
    image: ubuntu:22.04
    min_ready: 2        # Keep 2 containers ready
    max_total: 10       # Maximum 10 containers total
    health_check_interval: 30s

  - name: alpine-pool
    type: container
    provider: docker
    image: alpine:latest
    min_ready: 3
    max_total: 15
    health_check_interval: 30s
```

### 2. Initialize Boxy

The first run will create the database and encryption key:

```bash
# Create config directory
mkdir -p ~/.config/boxy

# Boxy will auto-generate encryption key on first run
# Key will be saved to: ~/.config/boxy/encryption.key
```

## Basic Usage

### Start the Boxy Service

```bash
# Start the service in foreground (for testing)
boxy serve

# Or run in background with systemd (see below)
```

### Create a Sandbox

```bash
# Create a sandbox with 2 Ubuntu containers for 1 hour
boxy sandbox create \
  --name "test-env" \
  --pool ubuntu-pool \
  --count 2 \
  --duration 1h
```

Example output:
```
Sandbox created: sb-abc123
Resources: 2 containers provisioned

Container 1:
  ID: res-xyz789
  Host: 172.17.0.2
  Username: root
  Password: <auto-generated>

Container 2:
  ID: res-def456
  Host: 172.17.0.3
  Username: root
  Password: <auto-generated>
```

### List Active Sandboxes

```bash
boxy sandbox list
```

### Get Sandbox Details

```bash
boxy sandbox get sb-abc123 --show-passwords
```

### Extend Sandbox Duration

```bash
boxy sandbox extend sb-abc123 --duration 2h
```

### Destroy a Sandbox

```bash
boxy sandbox destroy sb-abc123
```

### Check Pool Status

```bash
boxy pool list
```

Example output:
```
Pool Name       Ready  Allocated  Provisioning  Error  Total  Healthy
ubuntu-pool     2      2          0             0      4      ✓
alpine-pool     3      0          0             0      3      ✓
```

## Configuration

### Environment Variables

Override config file settings with environment variables:

```bash
# Storage
export BOXY_STORAGE_TYPE=sqlite
export BOXY_STORAGE_PATH=/path/to/boxy.db

# Logging
export BOXY_LOGGING_LEVEL=debug  # debug, info, warn, error
export BOXY_LOGGING_FORMAT=json  # text or json

# Encryption (optional - auto-generated if not set)
export BOXY_ENCRYPTION_KEY=$(cat ~/.config/boxy/encryption.key)
```

### Security: Encryption Key

Boxy automatically encrypts credentials (passwords) stored in the database.

- **First run**: Encryption key is auto-generated and saved to `~/.config/boxy/encryption.key`
- **Permissions**: Key file has `0600` permissions (owner read/write only)
- **Environment variable**: Override with `BOXY_ENCRYPTION_KEY` (base64-encoded 32-byte key)

**Important**: Backup your encryption key! Without it, you cannot decrypt stored passwords.

```bash
# Backup encryption key
cp ~/.config/boxy/encryption.key ~/boxy-encryption-key.backup

# Or set in environment for easier backup
export BOXY_ENCRYPTION_KEY=$(cat ~/.config/boxy/encryption.key)
```

## Running as a System Service

### Linux (systemd)

Create `/etc/systemd/system/boxy.service`:

```ini
[Unit]
Description=Boxy Sandbox Orchestration Service
After=docker.service
Requires=docker.service

[Service]
Type=simple
User=boxy
Group=boxy
WorkingDirectory=/home/boxy
ExecStart=/usr/local/bin/boxy serve
Restart=on-failure
RestartSec=5s
StandardOutput=journal
StandardError=journal

# Security
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/home/boxy/.config/boxy

[Install]
WantedBy=multi-user.target
```

Enable and start:

```bash
# Create boxy user
sudo useradd -r -s /bin/false boxy
sudo usermod -aG docker boxy

# Copy config
sudo mkdir -p /home/boxy/.config/boxy
sudo cp ~/.config/boxy/boxy.yaml /home/boxy/.config/boxy/
sudo chown -R boxy:boxy /home/boxy

# Enable service
sudo systemctl daemon-reload
sudo systemctl enable boxy
sudo systemctl start boxy

# Check status
sudo systemctl status boxy
sudo journalctl -u boxy -f
```

### macOS (launchd)

Create `~/Library/LaunchAgents/com.boxy.service.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.boxy.service</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/boxy</string>
        <string>serve</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardOutPath</key>
    <string>/tmp/boxy.log</string>
    <key>StandardErrorPath</key>
    <string>/tmp/boxy.error.log</string>
</dict>
</plist>
```

Enable:

```bash
launchctl load ~/Library/LaunchAgents/com.boxy.service.plist
launchctl start com.boxy.service
```

## Troubleshooting

### Docker Connection Issues

```bash
# Check Docker is running
docker ps

# Verify Boxy can connect to Docker
boxy pool list
```

If you see "Docker health check failed":
- Ensure Docker daemon is running
- Verify user has Docker permissions: `groups` should show `docker`
- On Linux: `sudo usermod -aG docker $USER && newgrp docker`

### Database Locked Errors

SQLite can have locking issues under heavy load:

```bash
# Check if multiple Boxy instances are running
ps aux | grep boxy

# Kill duplicates
kill <pid>
```

For production use with high concurrency, consider PostgreSQL (planned for v0.2.0).

### Orphaned Containers

If Boxy crashes, containers may be left running:

```bash
# List Boxy-managed containers
docker ps -a --filter "label=boxy.managed=true"

# Clean up orphaned containers
docker rm -f $(docker ps -aq --filter "label=boxy.managed=true")
```

### Encryption Key Lost

If you lose your encryption key, you **cannot** decrypt existing passwords:

1. Stop Boxy service
2. Delete database: `rm ~/.config/boxy/boxy.db`
3. Delete encryption key: `rm ~/.config/boxy/encryption.key`
4. Restart Boxy (will regenerate key and recreate database)

**This will destroy all existing sandboxes and resources.**

### Resource Pools Not Replenishing

Check pool status:

```bash
boxy pool list
```

If pools show errors:
- Check Docker image exists: `docker pull ubuntu:22.04`
- Check Docker has resources available (memory, disk)
- Check logs: `sudo journalctl -u boxy -f` (if running as service)

### Verbose Logging

Enable debug logging:

```bash
# Via environment variable
BOXY_LOGGING_LEVEL=debug boxy serve

# Or in config file (boxy.yaml)
logging:
  level: debug
  format: text
```
